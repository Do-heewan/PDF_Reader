# /study — 챕터별 학습노트 생성

너는 사용자가 **어떤 개발 서적이든**(영문·한글 불문) 한 챕터씩 학습하도록 돕는다.
대상 책은 `data/` 에 들어 있는 PDF이며, 결과 노트는 **항상 한국어**로 작성한다.
특정 책·기술 스택을 가정하지 말고, 책 제목·용어·코드 언어는 그 챕터 PDF에서 그대로 끌어온다.
이번 호출에서 정리할 챕터 번호는 `$ARGUMENTS` 다. (비어 있으면 어느 챕터인지 사용자에게 묻고 멈춘다.)

## 절차

1. **챕터 번호 정규화**: `$ARGUMENTS` 의 숫자를 두 자리로 만든다 (`3` → `03`, `12` → `12`).
   대상 폴더는 `chapters/chapter{NN}/`, 노트 파일은 `chapters/chapter{NN}/chapter{NN}.md`.

2. **원본 확인**: `chapters/chapter{NN}/source.pdf` 가 있는지 본다.
   - 없으면 노트를 만들지 말고, 다음을 안내하고 멈춘다:
     ```bash
     python scripts/extract_chapters.py            # 아직 분할 안 했을 때
     python scripts/extract_chapters.py --chapter {N}   # 해당 챕터만 다시 분할
     ```

3. **읽기**: `chapters/chapter{NN}/source.pdf` 를 Read 도구로 읽는다.
   - **텍스트뿐 아니라 그림·다이어그램·표까지** 해석한다. 아키텍처 그림은 학습의 핵심이다.
   - `chapters/chapter{NN}/source.txt` 가 있으면 텍스트 보조 참고용으로만 함께 본다
     (그림 정보가 없으므로 PDF가 항상 우선).

4. **노트 작성**: 아래 §규칙·§템플릿대로 `chapters/chapter{NN}/chapter{NN}.md` 를 Write 한다.
   - `pp.` 범위와 챕터 제목은 `toc.json` 에서 가져온다 (없으면 PDF 내용으로 추정).
   - 학습일은 오늘 날짜(`YYYY-MM-DD`).

5. **진행 갱신**: `progress.md` 가 있으면 해당 챕터 줄의 `☐` → `☑` 로 바꾸고 학습일 칸을 오늘로 채운다.
   (`progress.md` 가 없으면 건너뛴다.)

6. **마무리**: 사용자에게 노트 경로와 함께 **셀프 체크 3문항만** 보여주고,
   "보지 않고 답해본 뒤 블로그에 정리해보라"고 안내한다. **답은 알려주지 않는다.**

## 원칙

- **한 번에 한 챕터만.** 여러 챕터를 몰아서 생성하지 않는다. 한 챕터씩 꾸준한 학습이 목적이다.
- 책 문장을 그대로 옮기지 않는다. 핵심을 **자기 말로 재구성**한다 (저작권·학습 양면).
- **원문이 영어든 한국어든 노트는 한국어로** 쓴다.
- 코드·명령어·설정·매니페스트는 코드 블록으로. 식별자(클래스·함수·명령·리소스 이름)는
  백틱으로 감싼다. 책의 주제에 맞는 언어·도구를 그대로 쓰고 특정 스택을 가정하지 않는다.
- 군더더기·홍보성 표현 없는 **선언형 한국어 기술 문체**.
- 그림은 '무엇을 보여주는 그림인지 + 흐름'을 글/표/화살표로 재현한다.
- 다른 사람에게 챕터를 설명할 수 있을 정도의 분량. 불필요하게 늘이지 않는다.
- **셀프 체크는 질문만 남기고 답은 비운다. 회고는 사용자가 직접 채운다.**
  노트가 정답을 다 떠먹여주면 머리에 남지 않는다 — 이 두 섹션이 능동 학습 장치다.

## 학습노트 템플릿 (`chapter{NN}.md`)

````markdown
# {NN}. {챕터 제목}

> 학습일: {YYYY-MM-DD} · 원문 pp. {start}–{end}

## 한 줄 요약
이 챕터가 결국 무엇을 말하는지 한 문장.

## 선행 지식 / 직전 챕터와의 연결
이 챕터를 이해하려면 알아야 할 것, 앞 챕터에서 이어지는 맥락.

## 핵심 개념
### {개념 1}
- **정의**:
- **왜 필요한가**(어떤 문제를 푸는가):
- **동작 방식**:

### {개념 2}
...

## 주요 코드 / 명령어 / 설정
```text
(챕터 주제에 맞는 코드·명령·설정 예시. 책의 언어·도구를 그대로 사용)
```

## 다이어그램으로 이해하기
책의 핵심 그림이 보여주는 구조와 흐름을 글/표/화살표로 재구성.

## 한 줄 회고 — 직접 채우기
> (학습 후 본인이 작성: 가장 인상 깊었던 점 / 실무에서 쓸 곳)
````
